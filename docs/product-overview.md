# OwlAlpha 产品概述

## 1. 产品定位

OwlAlpha 是一个本地优先的 A 股 AI 智能分析系统，帮助用户将行情、新闻、公司资料和大模型分析能力整合到同一个可控的研究平台中。项目面向希望自建股票分析工具的个人投资者、研究人员和小型团队，强调可部署、可追踪、可复盘和可扩展。

与依赖云端托管的通用型产品不同，OwlAlpha 默认将数据、配置和分析结果保留在用户自己的运行环境中。`0.1` 版本仅支持接入用户自定义的 OpenAI 兼容 API，通过可配置的 `Base URL`、模型名和 API Key 完成分析调用。

## 2. 目标用户

### 2.1 个人投资者

- 需要构建自己的 A 股观察池与每日报告流程。
- 需要一个可长期积累历史结论与研究记录的系统。
- 希望使用自有 API Key 和自定义模型接口，控制接入方式与成本。

### 2.2 研究型用户

- 需要针对一组固定股票进行重复性分析。
- 需要追踪行情、新闻与历史报告之间的关系。
- 需要更稳定、结构化的研究辅助工具，而不是零散的脚本。

### 2.3 小型团队

- 需要在单机、内网服务器或家庭服务器中部署统一工具。
- 需要以较低维护成本完成本地化部署。
- 需要为后续协作、权限和更多数据接入预留基础架构。

## 3. 解决的问题

OwlAlpha 重点解决以下问题：

- 研究流程割裂：行情、新闻、AI 分析结果散落在多个工具中。
- 分析结论难复盘：缺少统一的历史报告与任务记录。
- 部署不可控：很多 AI 应用要求强依赖公有云和第三方服务。
- 扩展困难：个人脚本容易快速失控，难以演进成稳定系统。
- 使用门槛高：很多现有方案不适合普通用户进行长期本地维护。

## 4. 产品目标

### 4.1 V0.1 目标

- 提供可运行的 A 股股票池与分析任务管理能力。
- 建立完整的“行情 + 新闻 + AI 分析 + 每日报告留存”主链路。
- 提供后台登录能力，登录后统一查看分析相关内容。
- 支持 Docker Compose 一键部署。
- 支持用户自定义 OpenAI 兼容接口配置。

### 4.2 后续阶段目标

- 提升新闻处理和上下文构建能力。
- 支持更多分析模板与任务调度策略。
- 增加更完整的报告中心和权限能力。
- 在保持本地化部署特性的前提下，扩展系统的插件化与可定制能力。

## 5. 核心能力范围

### 5.1 股票池管理

- 维护 A 股股票基础信息。
- 创建和管理分组、自选列表、关注状态。
- 支持批量导入和批量启停分析任务。

### 5.2 行情与基础数据

- 拉取 A 股日线行情与基础报价。
- 支持单股行情摘要和报告上下文组织。
- 支持本地缓存和重复请求控制。

### 5.3 新闻聚合与处理

- 获取和股票相关的新闻资讯。
- 对新闻进行去重、清洗和时效过滤。
- 组织为可供大模型分析的上下文片段。

### 5.4 AI 分析与报告生成

- 对单只股票进行结构化分析。
- 输出风险提示、观察要点和摘要结论。
- 保存 Prompt 版本、模型信息和分析快照，支持后续复盘。
- 支持按日批量生成报告。

### 5.5 任务调度

- 支持手动发起分析任务。
- 支持按计划批量执行股票池分析。
- 记录任务执行状态、失败原因和耗时。

### 5.6 本地化部署

- 默认以本地数据库和本地文件卷保存数据。
- 支持使用 Docker Compose 完成整套部署。
- 支持通过后台配置自定义 OpenAI 兼容 API。

### 5.7 后台管理

- 后台需要登录后访问。
- 登录后集中查看股票池、任务、报告和系统配置。
- 不单独提供面向普通访客的前台站点。

## 6. 非目标范围

为了确保第一版聚焦，以下内容不属于当前阶段目标：

- 港股和美股分析
- 独立用户端前台
- 本地模型和 Ollama 接入
- 自动化交易与券商接入
- 复杂回测和量化策略执行
- 多租户 SaaS 产品化能力
- 企业级复杂组织协作流

## 7. 价值主张

OwlAlpha 的核心价值来自以下三个维度：

- 本地优先：用户可以尽可能将敏感数据、分析历史和模型接入过程掌握在自己的环境中。
- 可复盘：系统会保留任务、报告和关键上下文，帮助用户建立可追踪的研究闭环。
- 可扩展：系统不是一次性的分析脚本，而是可以逐步演进的产品化底座。

## 8. 成功标准

项目初期的成功标准包括：

- 用户可以在本地或服务器中通过 Docker Compose 成功部署。
- 用户可以登录后台并完成 OpenAI 兼容接口配置。
- 用户可以配置股票池并成功生成 A 股每日报告。
- 系统可以留存历史报告并展示任务执行记录。
